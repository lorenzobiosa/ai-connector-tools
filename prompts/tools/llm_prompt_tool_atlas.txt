Esegui comandi MongoDB Atlas CLI in modo sicuro tramite Azure Function.  
- Nessuna credenziale nel payload: API Key da Key Vault (Managed Identity).  
- Autenticazione: `api_key` (default), `profile` o `auto`.  
- Il comando viene passato tal quale: tutte le opzioni (es. `--output json`, `--projectId`, `--tier`) devono essere fornite dal chiamante.  
- Nessun flag aggiunto dal backend: il connettore Ã¨ generico e astratto.  
- Se il comando richiede `projectId`, deve essere incluso nel payload.  
- Codici HTTP: 200 (ok), 400 (errore CLI o parametri), 408 (timeout), 500 (errore imprevisto), 502 (problema infrastrutturale).  
- Output: `stdout`, `stderr`, `exit_status`, argv eseguito, timing, request_id.  
Esempio:  
`{"command":"clusters list --projectId <ID> --output json","auth_method":"api_key","timeout_sec":60}`
